# HMM/HSMM 分析器 — 开发路线评审与修正建议

- **日期**：2026-06-03
- **评审对象**：《HMM/HSMM 分析器高投入开发执行计划 v0.1》(2026-06-01)
- **前提**：本评审假定先前《代码审计报告》中的问题（HSMM 校准目标错配/horizon 泄漏、duration 删失、数据校验覆盖、宽度与自定义池口径等）**已修复并有回归测试守护**。因此本文不再讨论存量 bug，只评估**前向开发路线的方向、难度、缺陷与修正**。
- **性质**：方法论与优先级评审，非代码改动。

---

## 1. 总评

**主线方向是对的，基建也已经搭得不错。** 计划提出的主线——

```
causal baseline → sticky + confidence → HSMM 只读诊断 → duration hazard 校准 → BOCPD 突变 → decision engine
```

——把"复杂生成模型关进研究分支、用可校准的监督式模型承担实践判断"这条最关键的判断做对了。证据账本 + readiness gate（已实现）是真正的资产，它让"模型可以复杂，但展示必须分级"成为制度。

**但计划有四个结构性问题需要修正：**

1. **锚错了难度**：标的是"工程难度"，真正的约束是"样本外统计有效性"。多数阶段**好建、难验**。
2. **样本天花板被低估**：A股可用的独立 regime 段数远少于参数与切片需求，这是 HSMM/hazard/decision 的硬上限。
3. **模型动物园 → 多重检验过拟合**：多模型 × 多用途 × 多 horizon × 多状态，"打败基准就晋升"会系统性选中靠运气赢的模型。
4. **验收被收益锚定**，与"判断≠买卖"的定位自相矛盾；本系统的边际价值在**风险**，不在收益。

下面分别给难度评估、缺陷、修正建议和可达效果。

---

## 2. 各阶段难度与现状评估

打分 1=易，5=极难。**工程难度**=写代码与系统集成；**验证难度**=样本外能否站得住。

| 阶段 | 工程 | 验证 | 现状（假定已修复后） | 关键风险 / 判断 |
|---|---|---|---|---|
| 0 证据账本 + readiness gate | ★★ | ★ | 已完成 | 资产，维持即可 |
| 1 HMM 强化（confidence/稳定性/label对齐/churn） | ★★ | ★★★★ | 大部分已完成 | 收益型验收门可能不可达，应改锚风险 |
| 1+ Sticky / 厚尾 emission | ★★★★ | ★★★★ | 未做 | sticky 性价比高；厚尾(t/GH)工程重、边际小 |
| 2 HSMM 生命周期校准 | ★★★ | ★★★★★ | 已修目标/泄漏/删失 | **即使修完，细粒度校准仍受样本天花板限制** |
| 3 Duration Hazard（离散时间监督式） | ★★ | ★★★ | 有雏形 | **全计划最高性价比，应升为主退出引擎** |
| 4 BOCPD 结构突变 | ★★★ | ★★★★ | 未做 | A股噪声下易抖动/滞后；先做便宜的断点检测器 |
| 5 高级模型（VAR-HSMM/厚尾图/深度SSM） | ★★★★–★★★★★ | ★★★★★ | 未做 | 高投入低期望边际；严格关在研究分支 |
| 6 决策引擎 + abstain | ★★★ | ★★★★ | 未做（基建已具备） | 多信号融合是过拟合重灾区，需终测集纪律 |

**要点：**
- **工程难度低 ≠ 容易**。Stage 3 的 logistic/isotonic 几行就能写，但"在每个状态×horizon 上稳定通过校准门槛"才是真难点，而它取决于样本。
- **时间线是工程时间，不是验证时间**。Stage 2/3/4 的真实周期取决于"验证失败→重做"的迭代轮数，单人通常乘 1.5–2 倍。原计划的周/周数估计应理解为乐观下限。

## 3. 路线的结构性缺陷

**① 样本天花板（最根本）。** ~6 年 ~1500 交易日，单板块独立 regime 段常只有 10–30 个；再按 (状态 × horizon × age 桶) 切，每格样本是个位数。这是 HSMM 校准难、hazard 细分难、decision 融合难的**共同上限**。计划有 hierarchical shrinkage（好），但仍倾向"输出更多细分概率"，方向相反。

**② 多重检验 / 模型动物园。** 计划列了 baseline/sticky/heavy-tail/t/GH-HMM、VAR-HSMM、graphical HMM、deep SSM、logistic/GBM/RF hazard、BOCPD 等十余个模型，乘上 5 类用途、5 个 horizon、4+ 状态，构成巨大的多重比较面。readiness gate 只管**展示分级**，不管**模型选择的多重性**——"打败基准就晋升"在这么多次试验下必然产生假阳性。

**③ 验收锚错维度。** 多处验收门是"打败 RS20 / 等权"。但输出定位是 `risk_downshift / abstain`，主指标应是**风险调整与决策质量**，不是长多轮动的收益。团队自评 HMM 已是 Weak，继续用收益当门槛会逼出过拟合或长期卡死。

**④ Stage 2 与 Stage 3 冗余且顺序反了。** 两者产出同一个东西（校准退出倾向）。修生成式 HSMM 的 `p_exit` 高成本、低确定性；离散时间 hazard 低成本、天然可 isotonic 校准、能优雅降级。计划把 HSMM 修复排在 hazard 之前，等于先啃硬骨头。

**⑤ HMM 与 BOCPD 双 regime 检测器缺仲裁。** 计划说"BOCPD 显示 HMM 状态是否可靠"，框架好，但没规定**当 HMM 说"稳定 Trend"而 BOCPD 说"刚断裂"时决策引擎怎么办**。两个 regime 视角不一致是常态，必须有显式仲裁。

**⑥ 数据可信度只当一次性任务。** 即便存量问题已修，数据校验、覆盖率口径、复权/停牌处理应作为**常驻不变量**（全摄取路径校验 + CI 回归），否则一次上游变动就重新污染上层所有模型。

## 4. 修正建议

### 4.1 重排路线（核心修正）

把"先啃 HSMM、收益验收、堆模型"改成"先拿确定性高的、风险验收、强降级"：

```
修正后主线：

HMM causal baseline（已具备）
→ 数据可信度常驻不变量（全路径校验 + CI 回归）        ← 新增为持续项，而非一次性
→ Duration Hazard 作为主退出引擎（前置 Stage 3）       ← 与 HSMM 调换优先级
→ HSMM 降级为「状态分段 + 状态年龄」提供者             ← 只在 hazard 不足时才投入其校准
→ 便宜的多变量断点检测器 → 视情况再上 BOCPD
→ 精简 decision engine（强 abstain + 一次性终测集）
→ 高级生成模型（VAR-HSMM 等）永远留在研究分支
```

**关键调换**：让 **Duration Hazard 成为主生命周期信号**，HSMM 只提供因果状态路径和状态年龄。计划 §8 已经暗示了这条（"如果仍不稳定，则让 duration hazard 成为主生命周期信号"）——**把它从"兜底方案"升级为"默认主线"**。

### 4.2 验收口径修正

1. **改锚到风险**：主指标用最大回撤、尾部亏损(CVaR)、换手、风险暴露下降、"错过强势期"的代价；收益降为参考。对 `abstain` 的考核是"abstain 是否真的发生在后续高分散/高波动期"。
2. **一次性 held-out 终测集**：划出最后一段历史只在最终评审动一次；日常迭代只用 walk-forward 训练区。
3. **抗多重检验**：预注册主指标与主用途；用 deflated Sharpe / 多重检验校正；**模型晋升策略显式记录"试了多少个模型/参数"**并据此抬高门槛。
4. **purged + embargoed walk-forward**：在 horizon 周围做 purge/embargo，从制度上根除退出标签的 horizon 泄漏（即先前 H2 类问题的长期防线）。

### 4.3 方法替代（更省样本、更稳）

- **竞争风险（competing-risks）hazard 统一 `p_exit` 与 next-state**：现在"退出概率"和"退出去向"是两套、各自吃样本。把"退出到 Trend / 退出到 Stress / 退出到 Neutral"建成不同事件类型，一个模型同时给退出强度和去向，统计上更省样本、更自洽，且天然支持序数化输出。
- **regime 侧优先"sticky 先验 + 波动率标准化特征 + 强正则 Gaussian HMM"**，而不是先上厚尾/VAR-HSMM：能拿到厚尾收益大部分好处、成本极低。厚尾(t/GH) emission、VAR-HSMM、深度 SSM 维持在研究分支，用解释性报告而非主线 UI 评估。
- **突变侧先做"波动/相关/宽度 z-score 断点检测器"（便宜稳健），BOCPD 作为第二步**；并补 HMM↔突变检测器的**显式仲裁规则**（如：BOCPD break_warning 高时，强制把 HMM 状态 confidence 降级、decision 偏向 `risk_downshift/abstain`）。
- **输出粒度做减法**：与其追求每个 (状态×horizon×age) 的精确数值概率，不如默认输出**序数化倾向（低/中/高）**，仅在样本充足且通过校准门槛的少数切片才放出数值概率——这与样本天花板对齐，也与 readiness gate 的现有设计一致。

### 4.4 数据可信度作为常驻不变量

把"修过的数据问题"固化成**不会回归**的制度：
- 所有 OHLCV 摄取路径（板块/个股/指数/基准/全A）统一过同一套校验（含 `high≥low`、价在 `[low,high]`、非负量、重复日期、巨缺口）。
- 覆盖率、复权/停牌、涨跌停阈值口径写成断言测试纳入 CI。
- 把核心因果/回测/HSMM 测试纳入 CI（先前 CI 只跑了一小部分），让上层模型建立在"数据闸门有回归守护"之上。

## 5. 完成后的可达效果

分"现实可达 / 不太可能 / 真正有价值的终态"，便于管理预期。

**现实可达（做对了能拿到）：**
- 一个**诚实的市场风险环境面板**：regime + confidence + "状态不清晰会主动说不清晰" + 结构断裂早警 + 状态年龄/生命周期上下文。
- **方向性可靠的退出倾向**（经 hazard 校准）：回答"这个 regime/上涨是不是晚期了"，作为风险叠加层。
- **可审计的决策辅助控制台**：每个判断带 evidence packet、历史相似样本、abstain 理由——显著降低过度自信、支持人工判断、全程可追溯。
- 最可能的"赢点"：**在风险维度打败朴素基准**（回撤更小、尾部亏损更少、换手可控的风险过滤 / 仓位降级）。

**不太可能（请管理预期）：**
- 独立的、扣费后收益打败 RS20/等权的 **alpha 源**——边际价值在风险，不在收益。
- 每个状态/horizon 都可信的**精确数值 `p_exit`**——样本天花板让细粒度校准长期脆弱，**序数化倾向**会是多数切片的诚实上限。
- 在真正没见过的新 regime（OOD）里稳定有效——所以 abstain 是对的设计，但也意味着**最关键时刻系统常会说"我不知道"**。

**真正值得追求、且大概率可达的终态：**
> 一个有纪律、可审计的"**regime + 生命周期 + 突变**"风险上下文引擎，配**强 abstain 逻辑**——它是改善风险管理、防止过度自信犯错的**决策辅助**，不是印钞机。
>
> 而且这个终态主要靠 **Stage 3（hazard）+ Stage 4（突变）+ 一个精简的 Stage 6（decision）** 就能到达；**Stage 5 是可选研究线**，不在关键路径上。

---

## 6. 修正后的推荐主线与最小下一步

**推荐主线（一句话）：**
先把数据可信度变成常驻不变量 → **hazard 前置为主退出引擎、HSMM 降级为分段+年龄提供者** → 把所有验收**改锚到风险并用一次性终测集防多重检验** → 便宜断点检测器先于 BOCPD → 精简 decision engine 以 abstain 为核心 → 高级生成模型永远留研究分支。

**最小可执行下一步（4 步，全在关键路径上）：**
1. 建 Duration Hazard 的退出目标数据集 + 第一版 logistic/isotonic + age-bucket baseline + `insufficient_sample` 降级。
2. 用 **purged/embargoed walk-forward** 跑 hazard vs HSMM raw p_exit 的对照，产出 horizon×state readiness matrix。
3. 把验收主指标从"收益打败 RS20"改为**风险维度 + 决策质量**，并固定一段 held-out 终测集。
4. 把数据校验接到全摄取路径、把核心因果/HSMM 测试纳入 CI（常驻不变量）。

这 4 步完成后即可定型：**hazard 站得住 → 它当主生命周期信号，HSMM 退为结构解释**；hazard 也不足 → 说明是样本天花板问题，应进一步降低输出粒度而非堆模型。
